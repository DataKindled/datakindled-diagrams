Connecting to Snowflake

Choose the way in based on what you're trying to do

If you want to Explore data interactively, run queries, build apps
Snowsight
Snowflake's browser-based interface for managing data, running queries, and building applications. No installation required.
Browser-based, no setup — the interface used throughout this course
If you want to Script and automate SQL workflows
Snowflake CLI
Open-source command-line tool for running SQL and supporting developer workflows. Flexible, extensible, and designed for modern development practices.
Built for repeatable workflows — replaces the deprecated SnowSQL*
If you want to Build applications without leaving your editor
VS Code Extension
The Snowflake Extension for Visual Studio Code lets you connect to Snowflake, run SQL, and manage your data architecture directly in VS Code.
Stay in your development environment
If you want to Manage infrastructure as code
Terraform Provider
Manage Snowflake infrastructure as code by defining databases, warehouses, users, and roles in declarative HCL files instead of SQL or the web interface.
Declarative, version-controlled, repeatable environments
If you want to Connect a custom application programmatically
APIs & Drivers
Snowflake provides APIs and drivers for programmatic authentication, query execution, and resource management outside standard SQL interfaces.
Python, Java, .NET, Go, and more
If you want to Connect tools you already use
Third-Party Tools
Snowflake works with many industry-leading tools through a broad ecosystem of connectors, drivers, programming languages, and utilities.
dbt, Fivetran, Tableau, Power BI, and more
* SnowSQL has been deprecated. Snowflake recommends moving to Snowflake CLI.
In this course: demos are recorded in Snowsight — available immediately after account setup with no additional installation required. Developer tools and programmatic access become relevant in later chapters.
